# Die sieben Nicht-Verwechslungen — Spickzettel

Jede einzelne Regel hier steht schon in einem Doktrin-Dokument, aber
verstreut. Diese Seite bündelt sie an einem Ort, damit sie sich in einem
Blick prüfen lassen. Entstanden aus dem Abgleich mit einer externen Synthese
eines Freundes von Johannes (2026-07-04) — alle sieben deckten sich mit
bereits Bestehendem, nur die Bündelung fehlte.

1. **Schätzung ≠ Angebot.** Jede Kosten-Kamera zeigt eine Spanne mit
   Schätz-Vorschau-Label — nie ein Angebot. → `09_V0_SPEZIFIKATION.md`,
   KalkulationsEngine.
2. **Kamera ≠ Werkmessung.** Foto-Schätzung (±cm), AR (±1-2cm) und Laser (mm)
   sind drei Stufen, nie gemischt. → Drei-Toleranzen-Doktrin
   (`journal/IDEEN_TOPF.md`, 2026-07-03).
3. **RoomPlan ≠ Werkplan.** LiDAR-Aufmaß ist CAD-Rohbau-Vorlage, nicht die
   finale Werkzeichnung. → `16_SYSTEM_MATRIX.md` §1.
4. **AR-Vorschau ≠ CAD-Wahrheit.** Die begehbare 3D-Silhouette im Raum ist ein
   Verkaufsmoment, kein Ersatz für die echte CAD-Planung. → §1,
   `journal/IDEEN_CHRONOLOGISCH.md` #38.
5. **Wächter ≠ Fachprüfung.** Sicherheitsrelevante Wächter-Fälle (Elektro,
   Sicherungskasten, Trittschall) verweisen immer an den Fachbetrieb — die
   Kandidaten-Karte reicht dort nicht. → `16_SYSTEM_MATRIX.md` §6,
   Fachmann-Pflicht.
6. **Audio/Standort/Gesichter nur mit expliziter Freigabe.** Opt-in statt
   Opt-out, Fähigkeiten-Panel, 1-Klick-Widerruf — gilt auch für reine Reads.
   → §14, `03_BRIEFKASTEN_PROTOKOLL.md`.
7. **Alles gated, keine stillen Writes.** Jeder Schreibvorgang läuft über
   Karte→Bestätigung→Audit, nie automatisch. → `02_RAILS_UND_SICHERHEIT.md`.
