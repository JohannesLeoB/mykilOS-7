Das Killer-Feature

Der eine sichtbare Moment
Ein einziger „Zum Angebot"-Knopf im Projekt.

Aus dem am Projekt persistierten Warenkorb entsteht mit einem Klick eine beschriftete MYKILOS-Kalkulations-Vorschau (PDF) — Briefkopf, Positionstabelle, Netto / 19 % / Brutto, Angebotsnummer, Datum. Kein offizielles Angebot — das entsteht separat in sevDesk, mykilOS liefert nur die Vorstufe. Der Beweis, dass die ganze gebaute Kette durchgängig und wertig ist.

Fragebogen→ Kunde + Projekt→ Warenkorb (GRDB)→ Zum Angebot→ PDF→ Cash-Zeile

Fertig heißt: Schneider läuft einmal live durch

  1. Intake-Fragebogen ausgefüllt → Kunde + Projekt angelegt.
  2. Warenkorb als WorkBasket am Projekt in GRDB persistiert — überlebt Neustart.
  3. Aus dem Korb per einem Klick eine beschriftete Kalkulations-Vorschau (PDF), sichtbar im Angebote-Tab — kein offizielles Angebot, das entsteht separat in sevDesk.
  4. Cash-Widget zeigt die kalkulierte Summe genau dieses Auftrags.

Abgenommen live durch Johannes gegen Screenshots — nicht durch grüne Tests allein. Kein neues Subsystem: fast nur Gebautes zu einem Faden verdrahtet.

Morgen früh & der bewusste Verzicht

Die ersten 3 Schritte

  1. Roundtrip zuerst prüfen. Nicht die Migration „erstmals zünden" — sie läuft längst. Korb schreiben → App neu → identisch, plus Cold-Start gegen eine alte Row. Risiko ist Decode, nicht DDL.
  2. Block C beginnen. WorkBasketStore in bootstrap() instanziieren, speichere/lade/alle(projektNummer:) exponieren. Der eine Anschluss-Schritt, an dem alles hängt.
  3. Block D anlegen. Warenkorb→WorkBasket-Mapper mit dem Schneider-Fall als Testdaten, status = .kalkulation, projektNummer sauber geführt.

Was NICHT reinkommt

  • Klick-Picker / Positions-Extraktion — Flaggschiff, aber nicht nötig. Nach V11.
  • Abnabelung Phase 2/3 + Multi-Base — läuft über die bestehende Quelle.
  • sevDesk-Postbox Live-Schreibpfad — bleibt read-only. Cash nur kalkuliert.
  • Alerts-Kern, Morgen-Brief, Push, Delight — reife Politur, eigene Tiefe.
  • Volles DSGVO-Paket — nur Keychain-Härtung wird als Folge-Block vorweggenommen.
  • Generisches Pipeline-Modell — V10 braucht genau einen Port.
  • Mail senden, Clockodo-Buchung, ClickUp live — der Auftrag läuft ohne sie durch.
  • Zweiter PDF-Renderer — der einseitige reicht.

RAILS im Plan respektiert: main heilig (kein Push/Force) · externe Writes nur über gated Karte→Bestätigung→Audit · Sevdesk nur via Postbox · Airtable nie DELETE, Daniels Base nur lesen · Drive read-only (PDF bleibt lokal) · ClickUp nur Testspace + Ghost-Personas · Mail/Memos/Assistent nie kreuzlesbar · jede Integration per-User isoliert · jeder Alert braucht Datenschutz-Toggle · Kosten als Design-Kriterium · jede Bau-Einheit = kleiner Block + eigenes DMG + SAFETY.
