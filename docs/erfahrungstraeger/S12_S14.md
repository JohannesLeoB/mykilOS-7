## Wichtigstes Wissen

### Explicit Staging (Statut 12) — warum es existiert
Johannes hat immer uncommittete eigene Änderungen im Repo (z.B. `docs/IDEEN_UND_BACKLOG.md`). Ein `git add -A` würde diese versehentlich committen. Das ist kein theoretisches Risiko — es ist aus einem konkreten Incident entstanden. Niemals `git add -A`.

### Handoff-Dreifach-Pflicht (Statut 13) — warum sie existiert
Jede Session startet mit leerem Kontext. Ohne vollständigen Handoff (EREIGNISPROTOKOLL + CLAUDE.md + STARTPROMPT) startet die nächste Session blind. Das ist der einzige Mechanismus der das kollektive Gedächtnis am Leben hält.

### Kanonischer Ordner (Statut 10) — warum er existiert
Desktop-Worktrees (`~/Desktop/CLAUDE/`) sind Wegwerfkopien. Änderungen dort landen nie im richtigen Repo. Alles echte Arbeiten passiert im kanonischen Ordner.
