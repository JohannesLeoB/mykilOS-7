## Lessons learned

**S16-Konsultation war Gold wert.** Alle drei Fallstricke (Target-Grenzen, testbarer HTTP-Client, Kit-Sauberkeit) proaktiv erkannt — keiner hat gebissen.

**SourceKit-Diagnostics = Noise.** Wenn SourceKit Fehler meldet aber `swift build` grün ist: Compiler gewinnen lassen. Nicht ablenken.

**Validierung bricht bestehende Tests sofort.** Das ist kein Bug — das ist Feature. Wer schärfere Grenzen zieht, sieht sofort welche Tests auf altem Stand waren. In diesem Fall: 5 Tests mit `"appXYZ"` auf echte ID aktualisiert.

**Nicht-fataler Hook ist die richtige Entscheidung.** UserInfo-Abruf nach Login kann scheitern (Netzwerk, Scopes, Token-Race). Wenn er scheitert: leere User-Anzeige, kein Crash. Besser als OAuth-Flow blockieren.
